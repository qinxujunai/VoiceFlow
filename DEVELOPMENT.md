## 2026-05-23 — 项目启动

### 背景

用户想要一个本地语音转文字工具，解决三个痛点：
1. **联网依赖** — 现有工具（讯飞、微信输入法）网络差就废了
2. **收费** — Whisper API 按分钟收费，Wispr Flow $10/月
3. **中文支持差** — 很多英文工具对中文不友好

### 参考项目

- [Jarvis AI Assistant](https://github.com/akshayaggarwal99/jarvis-ai-assistant) — 开源语音助手，架构参考
- [awesome-llm-apps](https://github.com/Shubhamsaboo/awesome-llm-apps) — 100+ AI 应用模板，学习用

### 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 语音识别 | Sherpa-ONNX + SenseVoice | 阿里开源，中英日韩粤全覆盖，int8 量化 229MB，CPU RTF 0.026 |
| 音频采集 | sounddevice | 跨平台，简单好用 |
| 热键监听 | pynput | 支持全局热键 |
| 文字注入 | pyperclip + pyautogui | 模拟键盘粘贴，兼容所有应用 |
| UI | PyQt6 + HTML/CSS | 苹果风格毛玻璃效果，底部居中 |

## 2026-05-23（傍晚）— 方向调整：不做 AI 校对

### 竞品调研结果

深入研究了三个头部产品：

| 产品 | 核心方案 | 延迟 | 价格 |
|------|---------|------|------|
| **Wispr Flow** | 自研 ASR + 微调 Llama 云端校对 | ~700ms | $10/月 |
| **Speakly** | 系统级语音键盘，AI 清理口头禅 | 激活有卡顿 | 订阅制 |
| **Typeless** | 最强 AI 重写，重组内容结构 | 3-5 秒 | $30/月 |

### 重新思考痛点

跟用户深入讨论后，真正的痛点不是"缺 AI 校对"，而是三层：

1. **输入法卡顿** — 打一半切输入法崩溃、卡死，心态炸裂。语音直出不经过输入法，根治这个问题
2. **国外产品贵 + 中文差** — 对中文文化背景词汇（人名、公司名、领域术语）识别拉胯。SenseVoice 原生中文 + 热词注入，零成本
3. **怕丢** — 对着麦克风讲了一大段话，软件崩了什么都没留下。这是安全感问题，需要兜底机制

### 关键决策：不做 AI 校对

**为什么不做？**

- AI 校对（LLM 润色）本质是噱头——它增加延迟（+500ms~1s）、占用显存、引入复杂度
- 90% 的"润色"需求可以用零延迟的规则解决：去口头禅、补标点、中英文空格
- 剩下 10% 的真实需求（如"口语转书面语"）不是语音输入的核心场景
- 保持项目自包含——不需要额外配置模型，双击 `start.bat` 就能用

**替代方案：规则清理 + 热词兜底**

```
ASR 转写 → 规则清理（0ms延迟） → 粘贴
              │
              ├─ 去口头禅：嗯、啊、那个、就是说、然后然后
              ├─ 中英文空格：用Python开发 → 用 Python 开发
              ├─ 常见错字映射：七点云 → 奇点云
              └─ 基础标点：长句自动加逗号
```

热词从源头防止错误（让 ASR 别认错），规则在输出端清理（把漏网之鱼修掉）。两层防护，零额外延迟。

### 新增功能：剪贴板兜底

```
正常路径：转写成功 → 粘贴到光标 ✓
异常路径：粘贴失败 → 文字自动存入剪贴板
                    → 悬浮窗提示 "粘贴失败，文字已存入剪贴板"
                    → 同时写入本地历史文件（双重保险）
```

用户永远不用担心"说了半天白说了"。

### 新的技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| 语音识别 | Sherpa-ONNX + SenseVoice | 不变 |
| 文字后处理 | Python 正则规则 | 零延迟、零依赖、零显存 |
| 文字注入 | pyperclip + pyautogui | 不变，加兜底逻辑 |
| 热词管理 | hotword_loader + 动态添加 | 运行时可以加新词 |
| AI 校对 | ❌ 不做 | 噱头，规则够用了 |

## 学习笔记

### Sherpa-ONNX 是什么？

ONNX Runtime 的封装，专门用于语音识别。支持多种模型：
- SenseVoice（阿里开源，推荐）
- Whisper（OpenAI）
- Paraformer（阿里）
- NeMo（NVIDIA）

### SenseVoice 模型

- 支持语言：中文、英文、日文、韩文、粤语
- 模型大小：229MB（int8 量化）
- 识别速度：RTF 0.026（1秒音频 → 0.026秒处理）
- 准确率：中文 95%+（安静环境）

### 什么是 RTF？

Real-Time Factor，实时率。RTF < 1 表示比实时快。
- RTF 0.026 = 处理 1 秒音频只需 0.026 秒
- 意味着比实时快 38 倍

### 什么是热词？

特定领域的专业术语，比如：
- 产品名：VoiceFlow、Hermes
- 人名：菌子、秦徐俊
- 专业词：Sherpa-ONNX、SenseVoice

添加热词后，识别准确率会提高。

### 正则表达式是干什么的？

一套"描述文字长什么样"的暗号。在 VoiceFlow 里用于：
- 匹配并删除口头禅：`(嗯|啊|那个|就是说)+` → 删掉
- 中英文间加空格：`([一-鿿])([a-zA-Z])` → `\1 \2`
- 修复常见错字：`七点云` → `奇点云`

### 什么是 PyQt6？

Python 做桌面窗口的乐高积木。VoiceFlow 用它做：
- 无边框透明窗口（悬浮在屏幕底部）
- 里面嵌一个 WebView 浏览器，加载 HTML/CSS 画毛玻璃效果
- 为什么要这么绕？因为 PyQt6 原生控件做不出好看的毛玻璃和呼吸灯动画，HTML/CSS 可以
